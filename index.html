What I've worked on

I got into programming in 2016. I developed a web application for Continental to manage bulk raw materials. I used Ruby on Rails and I was totally convinced. It was my first professional experience and I was able to accomplish all goals due to its productivity.

In 2017, I started working at Talkdesk. I was part of the team that developed the first Developer Portal app. We used Ruby on Rails for the API and React for the frontend.

Later, I joined a small team that built Workspace, a microfrontend engine that coordinated 30 plus apps built by different teams and technologies. We used vanilla JavaScript, a lot of iframes and postMessage and a bit of React. This environment was available both on web and desktop without any code change from the teams building the apps. We used Electron for that.

Currently, I'm working at Shape Construction. We have a React PWA application backed by a Ruby on Rails API. My main focus has been in reducing the amount of state we manage in the frontend, migrating the app from Redux to React Query. This improved the application reliability and also the development experience. We are doing more with less code.

At the same time, I implement new endpoints on the API as well as extending existing ones. As an example, I helped migrating our Active Storage files to a different cloud provider without downtime. Also, I extended our existing comment feature to support commenting on other entities.
